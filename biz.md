1. Produktidee

Die Plattform simuliert den kompletten Bewerbungsprozess für Studenten und Berufseinsteiger.

Statt nur Interviewfragen zu üben, durchläuft der Nutzer eine realistische Pipeline:

CV Analyse
→ Recruiter Interview
→ Technical Interview
→ Coding / Case Challenge
→ Hiring Panel Decision

Der Nutzer erhält anschließend:

Feedback

Verbesserungsvorschläge

Score für seine Performance

Das Ziel ist, Studenten auf reale Bewerbungen vorzubereiten und gleichzeitig Talent sichtbar zu machen.

2. Hauptnutzer
   Kandidaten (primär)

Studenten oder Juniors, die sich auf Bewerbungen vorbereiten.

Sie können:

ihren Lebenslauf hochladen

Interview-Simulationen machen

Challenges lösen

Feedback erhalten

ihren Score verbessern

Firmen (sekundär, später)

Firmen können:

eigene Challenges erstellen

Kandidaten entdecken

Talent Leaderboards sehen

Universitäten (optional)

Universitäten können:

Campus-Lizenzen kaufen

das Tool in Career-Kurse integrieren

3. Core Features
   3.1 Bewerbungs-Simulation

Der Nutzer startet eine Simulation für eine Rolle, z. B.:

Backend Developer Internship
Frontend Developer Internship
AI / Data Internship

Die Plattform simuliert mehrere Stages.

3.2 CV Analyse

Input:

CV Upload
optional GitHub Link
optional Projektbeschreibung

Output:

Invite / Reject
Feedback
Verbesserungsvorschläge

Beispiel Analyse:

Skill Fit

Projektqualität

Klarheit der Beschreibung

3.3 Interview Simulation

Der Nutzer beantwortet Fragen.

Beispiele:

Warum interessiert dich diese Rolle?
Erkläre ein Projekt von dir
Wie würdest du X implementieren?

Antworten werden bewertet nach:

Kommunikation

technischer Tiefe

Struktur

3.4 Technical / Coding Challenge

Je nach Rolle gibt es Aufgaben.

Beispiele:

API Design Aufgabe
Debugging Aufgabe
System Design Frage

Optional:

Coding Challenge mit Score
3.5 Hiring Panel Simulation

Am Ende simuliert die Plattform eine Entscheidung.

Beispiel:

Recruiter Feedback
Tech Lead Feedback
Team Lead Feedback

Finales Ergebnis:

Score (z.B. 74%)
Invite / Reject Simulation
3.6 Quick Fix Loop

Wenn der Nutzer schlecht abschneidet:

System schlägt Verbesserungen vor.

Beispiele:

CV strukturieren
Projektrolle klar erklären
technisches Konzept genauer beschreiben

Der Nutzer kann danach:

Simulation erneut starten
4. Challenges & Leaderboards

Zusätzlich kann es Challenges geben.

Beispiele:

Backend Challenge
AI Engineering Challenge
System Design Challenge

Teilnehmer bekommen Scores.

Ein Leaderboard zeigt:

Top Kandidaten
5. Firmen-Challenges (optional später)

Firmen können eigene Aufgaben posten.

Beispiel:

Porsche Informatik Backend Challenge

Flow:

Firma erstellt Challenge
→ Studenten nehmen teil
→ Leaderboard entsteht
→ Top Kandidaten werden eingeladen
